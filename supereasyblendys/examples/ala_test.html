Super-Easy Blendy Backgrounds

by Matthew O'Neill

  • Published in: CSS, HTML and XHTML |

  • Discuss this article »

Recently, while trying to implement a few different navigation ideas that a designer had thrown my way, I became frustrated with my weak image editing skills. The design was gradient-heavy, so a traditional approach to navigation markup and styling would require a dozen or so background image slices due to varying colors and height requirements.

After spending a mortifying amount of time creating the images—I’m a programmer by trade, so anything more complicated than MS Paint gives me the willies—I had to take a step back and figure out a better way. What if, after finishing, I needed to tweak the height? Or, God forbid, the color palette? My head was going to explode if I had to open an image editor again, so the Super Easy Blendy Backgrounds technique was born.

The blendy way

Almost all the gradients in the design given to me were blended to white, so I figured that if I created a PNG that was blended from transparent to white, I could use that as a background image, and rely on the background-color style to provide the other half of the blend.

I fired up my trusty image editor and fumbled around until I had created two PNG images: a 100×100 px white-to-transparent blend, and a 100×100 px black-to-transparent blend (I didn’t need the black, but I wanted to see what it would look like).

My two transparent-blend images, built in The Gimp

Using those images, I quickly threw together the CSS, and voila! Beautiful blendy backgrounds.

Here’s the CSS. Note: We’re using the following color-based style definitions to make the example easier to understand, but we all know not to use presentation-specific words in our class names, right? Right.

.gradwhite {
  background-image: url(grad_white.png);
}
.gradblack {
  background-image: url(grad_black.png);
}
.box {
  border: solid orange 2px;
  float: left;
  height: 100px;
  margin: 1px;
  width: 175px;
}
.blue { 
  background-color: #2382a1; 
}
.green { 
  background-color: #4be22d; 
}
.pink { 
  background-color: #ff009d;
}

Here’s the markup.

<div class="box gradwhite blue"></div>
<div class="box gradwhite green"></div>
<div class="box gradwhite pink"></div>
<div class="box gradblack blue"></div>
<div class="box gradblack green"></div>
<div class="box gradblack pink"></div>

Here’s example one, which produces this result:

A PNG gradient used as a background

Hey, it’s not scaling!

After giving myself a congratulatory pat on the back, the first thing I tried to do was scale that div to the size that I wanted. And, as you’ve probably already figured out, the background gradient didn’t scale. Argh! I could force the background image to repeat horizontally or vertically (using repeat-x or repeat-y), but there was no way to scale the background. (See example two if you want to feel the pain.)

CSS backgrounds don’t scale with the HTML element.

CSS3 is going to implement a background-size attribute, but since CSS3 has an ETA of never, that’s no help now. So what do we do? Well, we find something that will scale. Like the img element. Instead of using a background to display the png blend, we can use an img element, and set the width and the height to 100%.

Here’s the new CSS (overwriting what we had before):

.gradwhite {
  background-image: url(grad_white.png);
}
.gradblack {
  background-image: url(grad_black.png);
}

.blue { 
  background-color: #2382a1; 
}
.green { 
  background-color: #4be22d; 
}
.pink { 
  background-color: #ff009d;
}
.gradwhite img, .gradblack img {
  background-color: transparent;
  height: 100%;
  left: 0px;
  position: absolute;
  top: 0px;
  width: 100%;
}
.box2 {
  border: solid orange 2px;
  float: left;
  height: 150px;
  margin: 1px;
  position:relative;
  width: 84px;
}

Here’s the markup.

<div class="box2 gradwhite blue">
  <img src="https://alistapart.com/grad_white.png"https://alistapart.com/>

</div>
<div class="box2 gradwhite green">
  <img src="https://alistapart.com/grad_white.png"https://alistapart.com/>
</div>
<div class="box2 gradwhite pink">
  <img src="https://alistapart.com/grad_white.png"https://alistapart.com/>
</div>
<div class="box2 gradblack blue">

  <img src="https://alistapart.com/grad_black.png"https://alistapart.com/>
</div>
<div class="box2 gradblack green">
  <img src="https://alistapart.com/grad_black.png"https://alistapart.com/>
</div>
<div class="box2 gradblack pink">
  <img src="https://alistapart.com/grad_black.png"https://alistapart.com/>

</div>

Successful scaling—woohoo!

The image is showing up on top of my text! What gives?

Now that we have these sweet blendy backgrounds, let’s try to put some text in them. We’ll can make the boxes scale when the user resizes text by using ems to specify the box height and padding; we’ll also add a text-size declaration to a body CSS rule so that the fonts play nice with the @em@s. You can follow the attempt in progress at example four.

Text showing up behind the PNG gradient

Looks like the absolutely positioned image is showing up in front of the text. To correct that, we can simply use the universal selector to select any descendents of .gradwhite, position them relatively (greater specificity could, of course, override this as necessary), and give them a z-index of 1. Giving the img a z-index of 0 then keeps it from popping up. While we’re at it we’ll add a little style to the paragraphs, just to set them off.

Here’s the CSS so far:

.blue { 
  background-color: #2382a1; 
}
.green { 
  background-color: #4be22d; 
}
.pink { 
  background-color: #ff009d;
}

.gradwhite img, .gradblack img {
  height: 100%;
  left: 0px;
  position: absolute;
  top: 0px;
  width: 100%;
  z-index:0;
}

.gradwhite * {
  position: relative;
  z-index: 1;
}

.gradwhite p {
  margin: 0px;
  padding: 3px;
}

.box3 {
  border: solid orange 2px;
  float: left;
  margin: 1px;
padding: 5px;
  position:relative;
  width: 256px;
}

Here’s the new markup.

<div class="box3 gradwhite pink">

  <img src="https://alistapart.com/grad_white.png"https://alistapart.com/>
  <p>This text is showing up in front of the image.</p>
</div>

Here’s example five, which produces this result:

Text correctly appearing in front of the PNG gradient

Do you realize none of this works in IE6?

Well, we all know that IE6 and prior versions don’t support PNG transparencies. And since IE7 won’t have the majority market share at least for a little while, you’re probably thinking that this is a great exercise in something your boss will never let you use, right? Well, IE has this great little thing called the AlphaImageLoader filter. Without going too much into that (which is explained in ALA’s “Cross-Browser Variable Opacity with PNG: A Real Solution“), the important parts are that you can load a PNG image and set the filter to “scale,” which eliminates the need for all the other workarounds we’ve used for Firefox.

Here’s the CSS to add. (Line wraps marked » —Ed.)

.gradwhite {
  filter: progid:DXImageTransform.Microsoft. »
AlphaImageLoader(src='grad_white.png', »
sizingMethod='scale');
}

And here’s the markup. (Don’t stare too long at the ugly inline styles—they’ll be gone in just a second.)

<div class="box gradwhite pink" style="width:256px; »
height:50px;">
    <p>
      Wow, maybe this will actually work!
    </p>

</div>

Here’s example six, which produces this result:

Technique working in IE6

Wrapping everything up

So how do we make this work in both IE and Firefox at the same time? Well, we can target our styles for Firefox, and then use the star-HTML hack to override them for IE 6. I’m using the star-HTML hack here more in the interest of clarity than anything else; I think it helps to see everything together, plus it’s easier to cut and paste. However, when I use this technique in a production environment, I prefer to use conditional comments to target browsers. After all, hacks are hacks, and they aren’t guaranteed to work in perpetuity. You can read more about conditional comments at Quirksmode.

While we’re at it, we’ll add one more star-HTML rule (position:static applied to the box class) to deal with an IE 6 bug that prevents links from working when we’re using z-index the way we’re using it.

At last, the final CSS. (Line wraps marked » —Ed.)

<style type="text/css">
.grad img {
  height: 100%;
  left: 0px;
  position: absolute;
  top: 0px;
  width: 100%;
  z-index: 0;
}
.box {
  border: solid orange 2px;
  float: left;
  margin: 1px;
  position: relative;
  width: 165px;
  padding: 5px;
}
.box * {
  margin: 0px;
  position: relative;
  z-index: 1;
}
* html .grad {
  filter: progid:DXImageTransform.Microsoft.AlphaImage
Loader (src='grad_white.png', 
sizingMethod='scale');
}
* html .grad img {
  display: none;
}
* html .box {
    position:static;
}
.blue { 
  background-color: #2382a1; 
}
.green { 
  background-color: #4be22d; 
}
.pink { 
  background-color: #ff009d;
}

</style>

<!--[if IE 7]>
<style type="text/css">

.box {
  border: solid red 2px;
  height:2.5em;
}
</style>
<![endif]-->

The final markup.

And finally, cross-browser Super Easy Blendy Backgrounds. (For the CSS and markup in one file, please see example seven.)

Super easy blendys

That’s it. Go forth and blend in peace.
